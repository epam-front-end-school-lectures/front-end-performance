0. FUNDAMENTALS

You are probably already know something

​0.1 Styles at Top


<html>
	<head>
		<link rel="stylesheet" href="css/style.css">
	</head>
	<body>

	</body>
</html>
  • ​Page renders progressively, stylesheet by stylesheet - page don't looking broken on initial load.
  • ​User have a nice looking page faster - that's our primary goals as FE Developers.

​​0.2 Scripts at Bottom


<html>
	<head>
		
	</head>
	<body>
		<script src="script.js"></script>
	</body>
</html>
  • ​​JS is blocking assets download: because it can change the page, and browsers may need to deal with that before it can move on to anything else.
  • ​​ JS files respects order in which they are included (not with async attr, anyways).

​​​0.3 Minify Your Styles


.hey .there .students {
  display: none;
}

.another .selector:before {
  display: block;  
  content: 'OMG';
}
123 B 

.hey .there .students{display:none}.another .selector:before{display:block;content:'OMG'}
89 B
  • ​​In average you gain 25-30% reduce in a filesize.

0.7 ​​​​​Content Delivery Networks (CDN)

Browser can handle only a several connections to a several server.

More servers = More simultanious downloads = Faster page load

0.8 Try Out Async


		<script async src="some.third.party.script.js"></script>

The script is downloaded asynchronously while the rest of the page continues to get parsed. The script is guaranteed to be executed as soon as the download is complete. Keep in mind that multiple async scripts will be executed in no specific order.

0.10 prefer <link> over @import


<link rel="stylesheet" href="css/styles.css">
					

@import url('style.css');
  • @import will work only if put on a top of a file, which is not always obvious
  • When you use the second option through an external stylesheet, the browser is incapable of downloading the asset in parallel, which can block the download of other assets.

0.11 Minimize Repaints and Reflows


var myList = document.getElementById("myList");

for (var i = 0, len = arr.length; i < len; i++) {
    
    myList.innerHTML += "<li>" + arr[i].title + "</li>"; //reflow - appending to element
    
}
In the above for loop, a reflow will be triggered for every iteration of the loop. 10 iterations cause 10 reflows. 

var constructedHTML = "";

for (var i = 0, len = arr.length; i < len; i++) {
    constructedHTML += "<li>" + arr[i].title + "</li>"; //no reflow - appending to string
}

document.getElementById("myList").innerHTML = constructedHTML; //reflow
In this scenario, the elements are being constructed within a string. Not a single reflow is created by the loop, as the DOM is not being altered. Only once the array has been completely looped through is the string then applied as the innerHTML of an object, causing the only reflow of the function.
